===============
 Version Notes
===============

1.66 -- August 9, 2010
======================

+ Updated VS2008notes:

  + Added a :ref:`Quickstart <quickstart>` section.

  + Reorganized now that it's possible to use the |leptonlib|
    :doc:`binaries for Windows zip file <downloading-binaries>` instead
    of having to build all the libraries.

  + Updated :doc:`build instructions <building-image-libraries>` for
    `libtiff 3.9.4`, `libjpeg 8b`, `libpng 1.4.3`, and `zlib 1.2.5`.

  + Added information on :doc:`Visual Studio 2010 compatibility
    <vs2010-notes>`.

+ Removed definitions of ``COMPILER_MSVC=1`` and ``USE_PSTDINT`` from
  all projects.

+ Changed the :gui:`leptonlib-1.66` Project to be more compatible with
  Visual Studio 2010.

  + For :gui:`Lib Debug` and :gui:`Lib Release` configurations reset
    linker and librarian :gui:`Output File` property to default to avoid
    VC++ 2010 problems.

  + Removed the conditional :gui:`Custom Build Step` and added an
    unconditional :gui:`Post-Build Event` to avoid VC++ 2010 bug where
    setting :gui:`Outputs` causes the linker to be invoked when building
    `leptonlib`. See `VC++ in VS2010 fails if there is a command to copy
    lib file in 'Custom Build Step'
    <http://connect.microsoft.com/VisualStudio/feedback/details/557340/vc-in-vs2010-fails-if-there-is-a-command-to-copy-lib-file-in-custom-build-step>`_
    for details.
